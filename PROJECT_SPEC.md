# StartWave 1.0 — Project Specification

## 1. Назначение версии 1.0

StartWave 1.0 — статическая цифровая стартовая страница для быстрого доступа к ежедневной информации, сервисам, развлечениям, игровым материалам и AI-инструментам.

Основной вопрос интерфейса:

> Что ты хочешь сделать сейчас?

## 2. Область реализации

Версия 1.0 должна предоставлять:

- главную страницу с понятной структурой;
- навигацию по существующим разделам;
- текущее время и дату;
- поиск через Яндекс;
- блок ежедневной информации;
- основные тематические направления;
- адаптивный интерфейс для компьютеров и мобильных устройств.

## 3. Что не входит в версию 1.0

На текущем этапе не реализуются:

- личный кабинет;
- регистрация и авторизация;
- AI-чат;
- сложная персонализация;
- хранение пользовательских данных;
- задачи, напоминания и заметки;
- получение реальной погоды из внешнего API.

## 4. Слоган версии 1.0

> **Твой день. Твои сервисы. Твои идеи.**

## 5. Структура главной страницы

Блоки должны располагаться в следующем порядке.

### 5.1 Header

- логотип `🌊 StartWave`;
- основная навигация;
- текущее время;
- текущая дата.

### 5.2 Hero

- название StartWave;
- краткое описание проекта;
- центральный поиск через Яндекс.

### 5.3 Блок «Сегодня»

- `🕒 Время` — показывает текущее локальное время;
- `📅 Календарь` — показывает текущую дату;
- `🌤 Погода` — содержит понятную заглушку до подключения погодного сервиса.

### 5.4 Блок «Основные разделы»

- `📜 Chronicles of Black Desert` — переход в раздел BDO;
- `🎬 Entertainment` — переход в раздел развлечений;
- `🔧 Services` — переход в раздел сервисов;
- `🤖 AI` — переход в раздел AI.

### 5.5 Блок «О проекте»

- кратко объясняет назначение StartWave;
- содержит утверждённый слоган или его смысл;
- не дублирует полное продуктовое видение.

### 5.6 Footer

- содержит год, автора и название проекта.

## 6. Принцип навигации

StartWave строится вокруг действий пользователя, а не вокруг простого списка сайтов.

Основные сценарии:

1. `🔍 Найти информацию`
   - поиск;
   - новости;
   - статьи;
   - знания.

2. `🎬 Отдохнуть`
   - фильмы;
   - сериалы;
   - музыка;
   - видео.

3. `🛠 Воспользоваться сервисом`
   - полезные инструменты;
   - почта;
   - карты;
   - повседневные сервисы.

4. `🤖 Использовать AI`
   - AI-инструменты;
   - помощь в работе;
   - обучение;
   - создание контента.

5. `🎮 Поиграть`
   - игровые платформы;
   - Steam;
   - Epic Games;
   - Battle.net;
   - Black Desert;
   - игровые сообщества.

Порядок блоков должен вести пользователя от универсальных действий к специализированным.

## 7. Правило структуры разделов

Каждый раздел должен отвечать на вопрос или намерение пользователя, а не просто показывать набор ссылок.

Пример:

Нежелательно:

```text
Развлечения: YouTube, Spotify, Кинопоиск, Twitch
```

Предпочтительно:

- Хочу фильм → Кино и сериалы;
- Хочу музыку → Музыка;
- Хочу видео → Видео.

## 8. Функциональные требования

### 8.1 Время и дата

- время и дата формируются на стороне браузера;
- Header показывает актуальные значения;
- карточки «Время» и «Календарь» показывают те же значения;
- данные обновляются без перезагрузки страницы;
- локаль отображения — `ru-RU`.

### 8.2 Поиск

- запрос отправляется в Яндекс;
- текст запроса передаётся через параметр `text`;
- результат открывается в новой вкладке;
- поле имеет доступную текстовую подпись.

### 8.3 Навигация

- все локальные ссылки ведут на существующие страницы;
- текущая страница обозначается через `aria-current="page"`;
- карточка Chronicles of Black Desert в версии 1.0 ведёт на `bdo.html` до появления отдельной страницы.

### 8.4 Погода

- в версии 1.0 допускается статическая заглушка;
- подключение внешнего погодного API относится к будущему этапу.

## 9. Требования к интерфейсу

- светлый синий океанический фон;
- стеклянные полупрозрачные карточки;
- мягкие световые волны;
- центральное расположение основного контента;
- современная системная типографика;
- единый ритм отступов;
- заметные заголовки карточек;
- hover-эффекты для интерактивных карточек;
- видимые focus-состояния для клавиатурной навигации;
- отсутствие визуальной перегрузки.

## 10. Адаптивность

- страница должна работать на ширине от `320px`;
- на узких экранах карточки перестраиваются в одну колонку;
- навигация остаётся доступной без горизонтальной прокрутки;
- поисковая форма адаптируется под ширину экрана;
- текст и элементы управления не должны обрезаться или накладываться друг на друга.

## 11. Доступность

- документ содержит один основной заголовок `h1`;
- заголовки секций образуют последовательную иерархию;
- все интерактивные элементы доступны с клавиатуры;
- фокус визуально заметен;
- элементы формы имеют подписи;
- идентификаторы элементов уникальны;
- декоративные символы не мешают экранным дикторам.

## 12. Технические ограничения

- проект использует статические HTML, CSS и JavaScript;
- основные стили подключаются из `assets/css/style.css`;
- основная логика подключается из `assets/js/script.js`;
- версия 1.0 не требует серверной части;
- локальные ресурсы должны использовать корректные относительные пути.

## 13. Критерии готовности StartWave 1.0

Версия считается готовой, если:

- все обязательные блоки присутствуют;
- часы и дата корректно отображаются в Header и карточках;
- поиск формирует корректный запрос Яндекса;
- отсутствуют битые локальные ссылки и ресурсы;
- консоль браузера не содержит ошибок JavaScript;
- страница не имеет горизонтальной прокрутки на поддерживаемых размерах;
- мобильная сетка работает;
- основные элементы доступны с клавиатуры;
- визуальный стиль соответствует требованиям версии 1.0.
