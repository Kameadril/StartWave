# Git шпаргалка StartWave 🌊

## 1. Начало работы с проектом

Перед выполнением команд Git нужно перейти в папку проекта StartWave.

### Windows

```powershell
cd C:\Projects\StartWave
```

### macOS

```bash
cd ~/Projects/StartWave
```

После перехода все следующие команды нужно выполнять внутри этой папки.

## 2. Проверка состояния

```bash
git status
```

Команда показывает текущее состояние проекта:

- **ветка** — над какой версией проекта ведётся работа, например `main`;
- **изменённые файлы** — какие файлы были добавлены, изменены или удалены;
- **готовность к коммиту** — какие изменения уже подготовлены, а какие ещё нужно добавить через `git add`.

Полезно выполнять `git status` перед началом работы и перед созданием коммита.

## 3. Просмотр изменений

```bash
git diff
```

Команда показывает изменения внутри файлов: какие строки были добавлены, удалены или исправлены.

Если появился экран с надписью `(END)`, нажми:

```text
q
```

Это закроет просмотр и вернёт обычную командную строку.

Для краткого списка изменённых файлов используй:

```bash
git diff --stat
```

Команда показывает имена файлов и количество изменённых строк без полного содержимого изменений.

## 4. Сохранение изменений

```bash
git add
```

`git add` подготавливает выбранные файлы к коммиту. Сам коммит эта команда ещё не создаёт.

Пример для одного файла:

```bash
git add PROJECT_SPEC.md
```

После команды желательно снова проверить состояние:

```bash
git status
```

## 5. Создание коммита

```bash
git commit -m "описание"
```

Коммит сохраняет подготовленные изменения в локальной истории проекта. В сообщении нужно коротко описать, что было сделано.

Пример для StartWave:

```bash
git commit -m "Restructure StartWave documentation"
```

Хорошее сообщение помогает позже понять содержание изменения без просмотра всех файлов.

## 6. Отправка на GitHub

```bash
git push
```

Команда отправляет локальные коммиты в удалённый репозиторий GitHub.

Сначала нужно создать коммит. Незакоммиченные изменения команда `git push` не отправляет.

## 7. Проверка после отправки

После `git push` снова выполни:

```bash
git status
```

Если все изменения сохранены и отправлены, ожидаемый результат содержит:

```text
nothing to commit, working tree clean
```

Фраза `working tree clean` означает, что в рабочей папке нет незакоммиченных изменений.

## 8. Частые предупреждения

Git в Windows может показать предупреждение:

```text
LF will be replaced by CRLF
```

Это не ошибка.

- `LF` — формат переноса строк, часто используемый в Linux и macOS;
- `CRLF` — формат переноса строк, обычно используемый в Windows.

Git сообщает, что при дальнейшей обработке файла переносы строк могут быть преобразованы под Windows. Обычно это не мешает работе проекта.

## 9. Рабочий процесс StartWave

Обычная последовательность работы:

1. Проверить состояние:

   ```bash
   git status
   ```

2. Посмотреть изменения:

   ```bash
   git diff
   ```

3. Подготовить нужные файлы:

   ```bash
   git add PROJECT_SPEC.md
   ```

4. Создать коммит:

   ```bash
   git commit -m "Restructure StartWave documentation"
   ```

5. Отправить коммит на GitHub:

   ```bash
   git push
   ```

Короткая схема:

```text
git status → git diff → git add → git commit → git push
```

Главное правило: сначала проверить изменения, затем сохранить их в коммите и только после этого отправлять на GitHub.
